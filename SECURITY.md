# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly by emailing **security@anthropic.com**.

Please do **not** open a public GitHub issue for security vulnerabilities.

## Scope

The following areas are in scope for security reports:

- **Subprocess injection** — command injection via CLI arguments or environment variables
- **Temporary file handling** — insecure creation, permissions, or cleanup of temp files
- **Information disclosure** — leaking API keys, session data, or sensitive content through logs or errors

## Response

We will acknowledge receipt within 48 hours and provide a timeline for a fix. We appreciate your help in keeping this project secure.
